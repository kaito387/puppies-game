## 1. 建模概述

## 4. 序列图

### 4.1 游戏启动序列

```mermaid
sequenceDiagram
    participant Player
    participant App
    participant SaveManager
    participant GameStore
    participant GameLoop

    Player->>App: 打开游戏
    App->>SaveManager: 检查是否有存档
    alt 存档存在
        SaveManager-->>App: 返回保存的GameState
        App->>GameStore: 初始化Store，加载状态
    else 无存档
        App->>GameStore: 创建新游戏
        GameStore-->>App: 返回初始GameState
    end

    App->>GameLoop: 启动游戏循环
    GameLoop->>GameLoop: 每200ms执行一次tick
    GameLoop->>GameStore: 调用tick()方法
    GameStore-->>GameLoop: 返回更新的GameState
    GameLoop->>App: 触发状态变化事件
    App->>App: 重新渲染UI
    Player-->>App: 看到游戏界面
```

### 4.2 购买建筑序列

```mermaid
sequenceDiagram
    participant Player
    participant UI
    participant GameStore
    participant Engine
    participant SaveManager

    Player->>UI: 点击购买建筑按钮
    UI->>GameStore: 调用buildBuilding()

    GameStore->>Engine: 验证资源充足
    alt 资源充足
        Engine-->>GameStore: true
        GameStore->>Engine: 执行buildBuilding()
        Engine->>Engine: 扣除资源<br/>增加建筑数量<br/>计算新的生产速度
        Engine-->>GameStore: 返回新的GameState
        GameStore->>GameStore: 更新state
        GameStore->>SaveManager: 自动保存
        GameStore-->>UI: 触发状态更新
        UI->>UI: 重新渲染ResourcePanel<br/>重新渲染BuildingPanel
        UI-->>Player: 显示建筑已购买<br/>显示资源更新
    else 资源不足
        Engine-->>GameStore: false
        GameStore-->>UI: 触发错误事件
        UI-->>Player: 显示"资源不足"提示
    end
```

### 4.3 研究技术序列

```mermaid
sequenceDiagram
    participant Player
    participant UI as ResearchPanel
    participant GameStore
    participant Engine
    participant SaveManager

    Player->>UI: 点击研究按钮
    UI->>GameStore: 调用research(techId)

    GameStore->>Engine: 验证前置条件
    alt 前置未满足
        Engine-->>GameStore: error
        GameStore-->>UI: 触发错误
        UI-->>Player: 提示前置技术未解锁
    else 前置已满足
        GameStore->>Engine: 验证资源充足
        alt 资源充足
            Engine-->>GameStore: true
            GameStore->>Engine: 执行research()
            Engine->>Engine: 扣除资源<br/>标记技术已研究<br/>应用技术效果
            Engine-->>GameStore: 返回新的GameState
            GameStore->>SaveManager: 自动保存
            GameStore-->>UI: 触发更新
            UI-->>Player: 显示研究成功<br/>显示新解锁内容
        else 资源不足
            Engine-->>GameStore: false
            GameStore-->>UI: 触发错误
            UI-->>Player: 提示资源不足
        end
    end
```

### 4.4 自动保存序列

```mermaid
sequenceDiagram
    participant GameLoop
    participant GameStore
    participant SaveManager
    participant localStorage

    loop 每5秒执行一次
        GameLoop->>GameStore: 获取当前GameState
        GameStore-->>GameLoop: 返回state
        GameLoop->>SaveManager: 调用save(state)
        SaveManager->>SaveManager: 序列化state为JSON<br/>添加版本号和时间戳
        SaveManager->>localStorage: 写入localStorage
        localStorage-->>SaveManager: 写入成功
        SaveManager-->>GameLoop: 保存完成
    end
```
